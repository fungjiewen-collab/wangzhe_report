# 王者荣耀世界市场监控日报 - 使用指南

## 📋 快速开始（5分钟完成设置）

### 第一步：初始化GitHub仓库

```bash
# 进入项目目录
cd wangzhe_report

# 运行初始化脚本
./init_github.sh
```

按照提示输入你的GitHub用户名和邮箱，脚本会自动：
- 初始化Git仓库
- 配置用户信息
- 创建初始提交

### 第二步：创建GitHub仓库

1. 访问 [https://github.com/new](https://github.com/new)
2. 仓库名称：`wangzhe_report`
3. 可见性：建议设为 **Public**（公开），这样任何人都可以访问报告
4. **不要**勾选 "Initialize with README"
5. 点击 "Create repository"

### 第三步：关联远程仓库并推送

```bash
# 替换 YOUR_USERNAME 为你的GitHub用户名
git remote add origin https://github.com/YOUR_USERNAME/wangzhe_report.git
git branch -M main
git push -u origin main
```

### 第四步：启用GitHub Pages

1. 进入你的GitHub仓库页面
2. 点击 **Settings** → **Pages**（左侧菜单）
3. 在 "Source" 部分：
   - Branch: 选择 `main`
   - Folder: 选择 `/ (root)`
4. 点击 **Save**

等待1-2分钟，GitHub Pages会自动部署。

### 第五步：访问报告

部署成功后，访问以下网址查看报告：
```
https://YOUR_USERNAME.github.io/wangzhe_report/
```

🎉 恭喜！你现在拥有了一个固定网址的市场监控报告！

## 🔄 日常更新流程

### 方式一：手动更新（推荐初期使用）

```bash
# 1. 编辑数据文件（JSON格式）
#    - data/official_events.json     # 游戏官方事件
#    - data/competitor_data.json     # 竞品动态
#    - data/user_feedback.json       # 用户需求

# 2. 生成新报告
python3 generate_report.py

# 3. 提交并推送
git add .
git commit -m "更新2026年3月31日数据"
git push

# 或使用一键部署脚本
./deploy.sh
```

### 方式二：自动化更新（进阶）

你可以配置定时任务（cron）每天自动执行：

```bash
# 编辑crontab
crontab -e

# 添加以下行（每天9:00执行）
0 9 * * * cd /path/to/wangzhe_report && python3 generate_report.py && ./deploy.sh >> /tmp/report_deploy.log 2>&1
```

## 📊 数据文件格式说明

### 1. 官方事件 (data/official_events.json)

```json
[
  {
    "date": "2026年3月30日",
    "content": "10:00正式开放昵称抢注；云游戏版本开放限时体验",
    "summary": "今日总结内容..."
  }
]
```

### 2. 竞品动态 (data/competitor_data.json)

```json
[
  {
    "date": "2026年3月30日",
    "competitors": [
      {
        "name": "竞品一：螃蟹",
        "content": "<h3>产品布局</h3><p>详细内容...</p>"
      }
    ]
  }
]
```

### 3. 用户需求 (data/user_feedback.json)

```json
[
  {
    "date": "2026年3月30日",
    "channels": [
      {
        "name": "小红书",
        "content": "<ul><li>用户反馈内容</li></ul>"
      }
    ]
  }
]
```

**注意：** `content` 字段支持HTML标签，可以使用 `<p>`, `<ul>`, `<li>`, `<strong>` 等格式化内容。

## 💡 常见问题

### Q1: GitHub Pages部署后访问显示404？

**A:** 请检查：
1. 是否已在Settings → Pages中正确配置Branch和Folder
2. 等待2-5分钟让GitHub完成部署
3. 确认index.html文件已推送到仓库根目录

### Q2: 如何修改报告样式？

**A:** 编辑 `generate_report.py` 中的CSS样式部分，然后重新生成报告。

### Q3: 能否自定义域名？

**A:** 可以！在GitHub Pages设置中添加自定义域名，并在仓库根目录创建 `CNAME`文件。

### Q4: 报告更新后同事看不到最新内容？

**A:** 
1. 确认已成功push到GitHub（检查GitHub仓库最后提交时间）
2. GitHub Pages有缓存，可能需要强制刷新浏览器（Ctrl+Shift+R 或 Cmd+Shift+R）
3. 等待1-2分钟让CDN刷新

### Q5: 如何添加新的日期数据？

**A:** 在对应的JSON文件中添加新对象，例如：

```json
[
  {
    "date": "2026年3月31日",  // 新日期
    "content": "新内容...",
    "summary": "新总结..."
  },
  {
    "date": "2026年3月30日",  // 旧日期保留
    ...
  }
]
```

脚本会自动按日期倒序排列，最新的显示在最前面。
